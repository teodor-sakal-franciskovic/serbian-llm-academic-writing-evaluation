# üìò Academic Writing Evaluation Rules

This document describes the evaluation rules used for assessing academic writing quality.
Each rule defines a clear instruction that must be satisfied in order to achieve a high score.
All of the definitions were originally written in Serbian language (and used like that in LLM prompts).

The rules are divided into:
- Global writing rules (apply to the entire document)
- Chapter-specific rules (apply to individual sections of the paper)

---

## üåç Global Writing Rules

| Rule Name | Rule Description |
|---------|------------------|
| Grammar and Spelling | The text must be grammatically correct and free of spelling errors. |
| Foreign Terms | Foreign words must be written in italics. If a term is translated into Serbian, the original foreign term must be provided in parentheses. |
| Abbreviations | When introducing an abbreviation, the full term must be stated first. Afterwards, only the abbreviation should be used (except in chapter titles). Abbreviations that are never used later are not allowed. |
| Argumentation | All claims must be supported by citations or justified by the results of the work. References must be embedded within the sentence and placed as close as possible to the claim they support. |
| Sentence Conciseness | Each sentence must express one and only one main point. |
| Sentence Clarity | Sentences must be unambiguous, easy to understand, and logically structured, without unnecessary complexity or vague phrasing. |
| Active and Passive Voice | Active voice should be the default. Passive voice should be used only when the agent is irrelevant, obvious, or when the focus is on the result or a general fact. |
| Sentence Opening | Sentences must not begin with conjunctions such as And, Or, or But, nor with numerals. |
| Sentence Content | Sentences must be formal in tone, avoid slang and jargon, and must not address the reader directly. |
| Tense Usage | Present tense is used for stating general facts, while past tense is used when describing the author's own results. |
| Comma Usage | Commas must be used before but and however, must not be used before and or or, are required in enumerations, for separating independent clauses, and are mandatory in appositions. |
| Punctuation | Exclamation marks should be avoided. Em dashes are used for inserted remarks, hyphens for compound words, and semicolons for pauses longer than a comma but shorter than a period. |
| Paragraph Conciseness | Each paragraph should cover one and only one topic. |
| Paragraph Organization | A paragraph must contain a topic sentence, supporting sentences that elaborate the idea, and a concluding sentence that summarizes or links to the next paragraph. |
| Terminological Consistency | The same term or phrase must be used consistently to refer to the same concept throughout the text. |
| Repetition | The text must not be repetitive. Concepts already presented elsewhere should not be repeated. |
| Unnecessary Details | The text must not include unnecessary details that are not essential for understanding the topic. |
| Relevance to the Topic | All content must be clearly related to the topic of the paper. |
| Verbosity | The use of generic or filler words should be avoided. |
| Use of Personal Pronouns | Personal pronouns should be avoided in order to maintain objectivity. |

## üìÑ Chapter-Specific Rules

### Problem Statement

| Rule Name | Rule Description |
|---------|------------------|
| Broader Problem Context | The broader problem must be clearly presented so its context and importance are immediately understood. |
| Core Concepts | Core concepts must be clearly defined and necessary for understanding the paper. |
| Significance of the Solution | The text must explain why the proposed solution is significant. |
| Positioning of the Narrow Problem | The paper must show how the specific problem fits into the broader context. |
| Description of the Paper Focus | The expected behavior of the solution must be clearly described. |
| Description of Solution Benefits | Beneficiary groups and their benefits must be clearly identified. |

---

### Theoretical Background

| Rule Name | Rule Description |
|---------|------------------|
| Domain Problem Description | The domain problem must be described in sufficient detail to derive solution requirements. |
| Requirements | A clear list of solution requirements must be provided. |
| Alternative Solutions | Alternative approaches must be briefly presented. |
| Justification of the Chosen Solution | The rationale for choosing the solution must be explicitly stated. |
| Description of Solution Concepts | Technologies and models must be clearly explained. |
| Defined Concepts | All technical terms must be clearly defined. |
| Excess Concepts | No unnecessary concepts should be included. |
| Mapping Concepts to Requirements | It must be clear how each requirement is realized. |

---

### Solution Description

| Rule Name | Rule Description |
|---------|------------------|
| High-Level Solution Overview | A conceptual overview of inputs, outputs, and purpose must be provided. |
| Initial System Structure | The main system components must be clearly presented. |
| Detailed Component Description | Each component must be described in logical order. |
| Precision | Enough detail must be provided to understand the procedure. |
| Level of Abstraction | Implementation details such as code should be avoided. |
| Use of Theory | Focus must be on how theory is applied, not explained. |
| Problems Encountered | Development problems must be explicitly stated. |

---

### Results

| Rule Name | Rule Description |
|---------|------------------|
| Result Conciseness | Desired and undesired outcomes must be clearly explained. |
| Level of Detail | Experiments must be described in sufficient detail for reproduction. |
| Tense in Results | Past tense must be used when describing results. |
| Results Structure | Results must be logically ordered and interpreted. |
| Results Discussion | Strengths, limitations, and comparisons must be discussed. |
| Final Paragraph | Results must be summarized and future work outlined. |
